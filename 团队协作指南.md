# 团队协作指南 - 上市公司财报"智能问数"助手

## 快速开始（新成员）

### 第一步：安装必要软件

```bash
# Windows 用户
1. 安装 Docker Desktop: https://www.docker.com/products/docker-desktop/
2. 安装 Git: https://git-scm.com/download/win

# 验证安装
docker --version
git --version
```

### 第二步：克隆项目

```bash
git clone <项目仓库地址>
cd 泰迪杯
```

### 第三步：配置环境

```bash
# 复制环境变量模板
cp .env.example .env

# 编辑 .env 文件，填入你的 API key
# Windows: 用记事本打开
# 或直接在 VS Code 中编辑
```

**.env 必填项：**
```ini
# 数据库密码（团队统一）
MYSQL_PASSWORD=fin_pass123

# LLM API 配置
ANTHROPIC_BASE_URL=https://your-api-endpoint.com
ANTHROPIC_AUTH_TOKEN=your-api-key-here
```

### 第四步：启动开发环境

```bash
# 一键启动所有服务
docker compose up -d

# 查看运行状态
docker compose ps

# 查看应用日志
docker compose logs -f app
```

### 第五步：访问系统

- **Streamlit 前端**: http://localhost:8501
- **MySQL 数据库**: localhost:3306

## 开发流程

### 1. 每天开始工作前

```bash
# 拉取最新代码
git pull origin develop

# 确保服务运行
docker compose ps
```

### 2. 创建功能分支

```bash
# 从 develop 创建新分支
git checkout develop
git checkout -b feature/your-feature-name

# 例如：
git checkout -b feature/text-to-sql-module
```

### 3. 开发和提交

```bash
# 添加修改的文件
git add <文件名>

# 提交（遵循提交规范）
git commit -m "feat: 实现 Text-to-SQL 核心功能"

# 推送到远程
git push origin feature/your-feature-name
```

### 4. 创建 Pull Request

在 GitHub/GitLab 上：
1. 选择你的功能分支
2. 点击 "New Pull Request"
3. 目标分支选择 `develop`
4. 填写 PR 描述，说明改动内容
5. 等待团队成员 Review

## Git 提交规范

### 提交格式

```
<type>: <简短描述>

[可选的详细说明]
```

### Type 类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `feat` | 新功能 | `feat: 添加多轮对话支持` |
| `fix` | 修复 Bug | `fix: 修复 PDF 解析乱码问题` |
| `docs` | 文档更新 | `docs: 更新 README 安装说明` |
| `style` | 代码格式（不影响功能） | `style: 格式化代码` |
| `refactor` | 重构 | `refactor: 重构数据库连接池` |
| `test` | 测试相关 | `test: 添加数据校验测试` |
| `chore` | 构建/工具配置 | `chore: 更新 Docker 配置` |

### 好的提交示例

```bash
feat: 实现财据数据自动入库功能

- 添加 PDF 表格解析功能
- 支持资产负债表、利润表、现金流量表
- 数据校验和清洗

fix: 修复 Streamlit 前端中文乱码

- 设置文件编码为 UTF-8
- 添加编码声明
```

### 不好的提交示例

```bash
# 太模糊
update code

# 太冗长
我今天花了 3 个小时修复了一个该死的 bug 现在终于好了

# 没有意义
asdfasdf
```

## 故障排查

### 问题 1：端口被占用

**错误信息**: `Bind for 0.0.0.0:8501 failed: port is already allocated`

**解决**:
```bash
# 修改 .env 中的端口
APP_PORT=8502
MYSQL_PORT=3307
```

### 问题 2：数据库连接失败

```bash
# 检查 MySQL 是否启动
docker compose ps mysql

# 查看 MySQL 日志
docker compose logs mysql

# 重启 MySQL
docker compose restart mysql
```

### 问题 3：代码变更不生效

```bash
# 重启应用容器
docker compose restart app

# 或重新构建
docker compose build
docker compose up -d
```
